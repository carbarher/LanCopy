FECHA: 15 Nov 2025 - 11:00 AM

PROBLEMAS DETECTADOS:
─────────────────────────────────────────────────────────────

1. ❌ La cola de descargas se iniciaba automáticamente al arrancar
   - Las descargas comenzaban sin que el usuario lo pidiera
   - Aparecían mensajes de reintentos y búsquedas en el log

2. ❌ Las descargas restauradas NO se mostraban en la grilla
   - Solo se agregaban a downloadQueue (memoria)
   - El usuario no podía ver qué descargas había pendientes


CAUSA RAÍZ:
─────────────────────────────────────────────────────────────

En LoadDownloadQueueAsync() (líneas 15246-15250):

```csharp
// ❌ ANTES (INCORRECTO):
if (loaded > 0)
{
    Log($"📂 Cola de descargas restaurada: {loaded} tareas");
    
    // Iniciar gestor de descargas AUTOMÁTICAMENTE
    if (!downloadManagerRunning)
    {
        _ = Task.Run(async () => await StartDownloadManager());
    }
}
```

Además, las descargas solo se agregaban a downloadQueue,
NO a la grilla visual (lvDownloads).


SOLUCIÓN IMPLEMENTADA:
─────────────────────────────────────────────────────────────

1. ✅ Eliminado inicio automático del gestor (líneas 15264-15267)
   - Ahora el usuario debe presionar "▶️ Reanudar" manualmente
   - Las descargas NO comienzan solas

2. ✅ Agregadas descargas a la grilla visual (líneas 15234-15249)
   - Cada descarga restaurada se muestra en lvDownloads
   - Con color según estado
   - Con progreso si estaba parcialmente descargado

3. ✅ Actualización inmediata del progreso (línea 15265)
   - Muestra "X/Y (Z%)" correctamente al inicio
